# Personal Knowledge Ops

这是一个用于管理个人知识、个人技能、个人区域信息、外部领域内容和副业项目的总控 skill。

## 首次使用

在会话里输入：

```text
@personal-knowledge-ops sync 初始化我的个人知识运营系统
```

它会自动创建以下中文目录：

- `个人知识`
- `个人技能`
- `个人区域`
- `外部领域内容`
- `副业`

其中 `副业` 下还会创建：

- `已确认可尝试`
- `确认不做`
- `建议的新项目`

## 日常使用

统一入口：

```text
@personal-knowledge-ops sync
```

你可以追加具体说明：

```text
@personal-knowledge-ops sync 我新增了一个技能：可以做健身新手训练计划
@personal-knowledge-ops sync 我这周试了一个副业项目，有2个咨询
@personal-knowledge-ops sync 我新增了几个知识文件，顺便扫一遍外部机会
```

## 这个 skill 会自动做什么

- 检查新增资料
- 归档和摘要知识内容
- 更新技能卡和技能索引
- 更新个人区域信息与个人汇总
- 同步副业实验进展
- 联网扫描外部相关机会
- 匹配个人能力与外部机会
- 更新副业候选

## 使用边界

它不能凭空知道你真实做了什么副业实验、结果怎样。

这部分仍然需要你在会话里主动补一句，例如：

- 我试了产品经理简历辅导，有2个咨询
- 我发了深圳营养内容，没有转化
- 我做了一个训练计划测试，有人愿意继续问

## 文件

- `SKILL.md`：skill 主规则
- `README.md`：使用说明
- `EXAMPLE-TREE.md`：初始化目录示例
- `VERSION.md`：当前版本
- `CHANGELOG.md`：更新记录
- `REPO-INFO.md`：仓库和安装链接信息
- `templates/`：固定模板，包括个人汇总、个人知识目录表

## 更新方式

当前推荐的更新方式不是自动覆盖，而是：

1. skill 运行时检查当前版本
2. 如果发现仓库里有新版本，提醒用户更新
3. 用户重新执行一次安装链接

示例：

```text
帮我安装这个skill：https://github.com/yourname/my-codex-skills/tree/main/personal-knowledge-ops
```

## 当前项目定位

这个项目是一个个人知识库工作流 skill，不是传统应用程序仓库。

它的核心职责是帮助你在长期对话中持续维护：

- `个人知识/`
- `个人技能/`
- `个人区域/`
- `外部领域内容/`
- `副业/`
